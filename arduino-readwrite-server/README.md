# Arduino Serial API Server (Rust)

Arduino の Serial 通信（read / write）を **HTTP API 経由で操作可能にする Rust 製 API サーバ**です。

USB 接続された Arduino と通信し、

- API → Arduino（Serial write）
- Arduino → API（Serial read）

を中継します。

---

## 🚀 特徴

- axum + tokio による高速・軽量 API サーバ
- Serial ポート設定を **環境変数で変更可能**
- API Key によるアクセス制御
- Docker / devcontainer 対応
- メモリリングバッファ + WebSocket 双方向通信
- ヘルスチェックエンドポイントあり

---

# 🔐 認証

`/read` と `/write`、`/logs`、`/ws` は **APIキー必須**です。

HTTPヘッダに以下を付与してください。

```
x-api-key: your-secret-key
```

`/`（ヘルスチェック）は認証不要です。

---

# 📡 API仕様

---

## GET /

ヘルスチェック用エンドポイント。

### Response

```
ok
```

---

## POST /write

Arduino の Serial にデータを書き込みます。

### Headers

```
Content-Type: application/json
x-api-key: your-secret-key
```

### Request Body

```json
{
  "data": "LED_ON\n"
}
```

### Response

```
OK
```

---

## GET /read

Arduino から最後に受信した Serial データを取得します。

### Headers

```
x-api-key: your-secret-key
```

### Response

```json
{
  "data": "OK\n"
}
```

---

## GET /logs

メモリリングバッファに保持された最新ログを取得できます。

### Headers

```
x-api-key: your-secret-key
```

### Query

| パラメータ | デフォルト | 説明 |
|------------|------------|------|
| limit      | 100        | 取得件数（最大） |

### Response

```json
[
  "OK",
  "LED_ON",
  ...
]
```

---

## WebSocket /ws

双方向通信が可能です。

- Arduino → WebSocket クライアントにリアルタイム配信
- WebSocket → Arduino への送信も可能

### Headers

```
x-api-key: your-secret-key
```

# ⚙ 環境変数設定

`.env` ファイルまたは環境変数で設定可能です。

| 変数名 | デフォルト | 説明 |
|--------|------------|------|
| SERIAL_PORT | /dev/ttyUSB0 | Serialポート |
| BAUD_RATE | 9600 | ボーレート |
| SERVER_ADDR | 0.0.0.0:3000 | サーババインド先 |
| API_KEY | changeme | APIアクセスキー |
| LOG_BUFFER_SIZE | 1000 | メモリリングバッファ最大件数 |

---

## 📄 .env サンプル

```env
SERIAL_PORT=/dev/ttyUSB0
BAUD_RATE=9600
SERVER_ADDR=0.0.0.0:3000
API_KEY=super-secret-key
LOG_BUFFER_SIZE=1000
```

### Serial Write

```
curl -X POST http://localhost:${SERVER_ADDR##*:}/write \
  -H "Content-Type: application/json" \
  -H "x-api-key: ${API_KEY}" \
  -d '{"data":"LED_ON\n"}'
```

```
  -H "x-api-key: ${API_KEY}"

```

```
  --device=/dev/ttyUSB0 \
  -e SERIAL_PORT=/dev/ttyUSB0 \
  -e BAUD_RATE=9600 \
  -e API_KEY=super-secret-key \
  -e LOG_BUFFER_SIZE=1000 \
  -p 3000:3000 \
  arduino-serial-api
```

# ⚠ macOS / Windows の注意

Docker Desktop では USB デバイスのパススルーに制限があります。

- macOS → `/dev/cu.*` は直接マッピング不可な場合あり
- Windows → WSL2 経由の設定が必要

開発用途では **ローカル実行を推奨**します。

# 🧪 Arduino サンプルコード

```cpp
void setup() {
  Serial.begin(9600);
}

void loop() {
  if (Serial.available()) {
    String cmd = Serial.readStringUntil('\n');
    if (cmd == "LED_ON") {
      Serial.println("OK");
    }
  }
}
```
